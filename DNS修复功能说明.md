# DNS污染检测与修复功能说明

## 功能概述

新增的DNS污染检测与修复功能可以自动检测GitHub相关域名的DNS污染问题，获取最新的可用IP地址，并更新系统的hosts文件以恢复正常访问。

## 支持的域名

- `github.com` - GitHub主站
- `raw.githubusercontent.com` - GitHub原始文件
- `github.github.io` - GitHub Pages
- `api.github.com` - GitHub API

## 功能特性

### 1. DNS污染检测
- 使用Google DNS (8.8.8.8)进行解析测试
- 识别污染IP地址（内网IP、127.x.x.x等）
- 检测DNS解析失败情况

### 2. 多源IP获取
- **ipaddress.com** - 专业IP地址查询服务
- **站长之家 (chinaz.com)** - 国内知名的站长工具
- **备用IP列表** - GitHub官方常用IP地址

### 3. 智能IP选择
- 测试每个IP的连通性
- 测量延迟（ping时间）
- 选择延迟最低的可用IP
- 验证IP的有效性

### 4. 系统文件更新
- 自动备份原有hosts文件
- 移除旧的GitHub相关条目
- 添加最新的IP地址映射
- 保留原有配置信息

### 5. DNS缓存刷新
- 自动识别系统类型
- 执行相应的DNS缓存刷新命令
- 支持CentOS/RHEL/Fedora/Ubuntu/Debian

### 6. 修复验证
- 测试GitHub访问是否恢复正常
- 显示修复结果
- 提供后续操作建议

## 工作原理

### 检测阶段
1. 使用dig命令通过Google DNS解析GitHub域名
2. 检查返回的IP地址是否正常
3. 识别常见的污染IP模式

### IP获取阶段
1. 从在线服务获取最新的GitHub IP地址
2. 如果获取失败，使用预置的备用IP列表
3. 收集多个候选IP地址

### 测试阶段
1. 使用ping测试每个IP的连通性
2. 测量延迟并选择最优IP
3. 验证IP地址的有效性

### 修复阶段
1. 备份当前的/etc/hosts文件
2. 创建新的hosts文件，移除旧的GitHub条目
3. 添加新的IP地址映射
4. 刷新系统DNS缓存

### 验证阶段
1. 测试GitHub访问是否正常
2. 显示修复结果
3. 提供使用建议

## 故障排除

### 常见问题

#### 1. 无法获取IP地址
- **原因**: 网络连接问题或在线服务不可用
- **解决**: 检查网络连接，脚本会自动使用备用IP

#### 2. 修复后仍无法访问
- **原因**: DNS缓存未完全刷新或IP已失效
- **解决**: 重启网络服务或手动刷新DNS缓存

#### 3. 权限不足
- **原因**: 未使用root权限运行
- **解决**: 使用sudo或以root用户运行脚本

#### 4. 依赖工具缺失
- **原因**: 系统未安装dig、curl等工具
- **解决**: 脚本会自动安装必要依赖

### 手动检查
```bash
# 检查GitHub域名解析
dig github.com @8.8.8.8

# 检查hosts文件
grep -i github /etc/hosts

# 测试GitHub访问
curl -I https://github.com
```

## 技术实现细节

### DNS污染检测算法
1. 使用权威DNS服务器（8.8.8.8）进行解析
2. 检查返回的IP地址是否符合公网IP规则
3. 排除保留IP地址段和私有IP地址

### IP获取策略
1. 优先使用在线服务获取最新IP
2. 多个来源增加成功率
3. 预置备用IP作为最后保障

### 连通性测试
1. 使用ping测试基本连通性
2. 测量延迟作为选择依据
3. 超时处理防止长时间等待

### 系统兼容性处理
1. 识别不同的Linux发行版
2. 使用相应的DNS缓存刷新命令
3. 处理不同的服务管理方式
